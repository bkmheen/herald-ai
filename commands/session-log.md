이번 세션에서 수행한 작업을 정리한 **개발기록 마크다운 파일**을 생성하십시오.
이 기록은 "어떤 작업이, 언제, 어떻게 진행되었는지"를 나중에 확인하는 용도입니다.

## 0. 먼저 수집

- **작업 디렉토리**(가장 중요): `pwd` 로 현재 절대경로를 구하고, git 저장소면 `git branch --show-current` 와 이번 세션 커밋(`git log --since=... --date=format:'%m-%d %H:%M' --pretty=format:'%cd | %s'`)으로 시각·내용을 보강하십시오.
- **세션 작업 내역**: 대화 전체를 되짚어 이번 세션에 실제로 한 일을 **일자·시간 순서**로 정리하십시오(커밋 시각을 앵커로). 추측 금지 — 실제 한 일만.

## 1. 파일명·위치

- 위치: 환경변수 `HERALD_LOG_DIR` 가 설정돼 있으면 그 경로, 없으면 `~/Desktop/`.
  (셸에서 `echo "${HERALD_LOG_DIR:-$HOME/Desktop}"` 로 대상 디렉토리를 먼저 확인하십시오. 없으면 생성.)
- 파일명: `{YYMMDD}-{요일} [{작업디렉토리 basename}]-{HHMM} 세션 작업기록.md`
  - `[{작업디렉토리 basename}]` 부제목이 어느 프로젝트의 기록인지 식별합니다 — 예: `herald-ai` → `[herald-ai]`.
  - `{HHMM}` 은 종료(생성) 시각. **여러 날에 걸치면 종료일 `{끝MMDD}`** 로 대신 표기(예: 06-13~06-14 → `-0614`).
  - 예(2026-06-13~14, herald-ai): `260613-토 [herald-ai]-0614 세션 작업기록.md`
  - 같은 이름 재생성 시 ` (2)` 등으로 덮어쓰기 회피.

## 2. 문서 구조 (반드시 이 순서)

```
# {제목} — 세션 작업기록

> **작업 디렉토리**: `{절대경로}`        ← 맨 앞·가장 중요
> **브랜치 / 저장소**: {git 정보(있으면)}
> **일시**: {시작 일시} ~ {끝 일시}
> **버전/커밋**: {버전 변화·커밋 수(있으면)}

## 요약 (개괄)
- 시간대별 개괄. 예: `20:11~22:00 — 1. A 작업 · 2. B 작업 · 3. C 작업`
- 몇 시부터 몇 시까지 무엇을 했는지 한눈에. (번호 매김 1./2./3. 권장)

## 작업 목록 (일자·시간별)
1. ({MM-DD HH:MM}) {작업명}
2. ({MM-DD HH:MM}) {작업명}
   … (시간 순)

## 세부 내용
### 1. {작업명}
- 배경/증상/목적 → 조치 → 검증/결과(테스트·커밋 해시 등)
### 2. {작업명}
…
```

## 3. 작성 원칙

- **작업 디렉토리**와 **요약**을 반드시 맨 앞에. 작업 디렉토리는 이 기록이 어느 프로젝트의 것인지 식별하는 핵심 정보.
- 요약은 개괄식(시간대 + 무엇을). 세부는 항목별로 "무엇을 왜 어떻게 + 검증/커밋".
- 사실만. 커밋이 있으면 해시·버전을 함께. 미해결·후속 과제도 명시.

## 4. 마무리

- 파일을 쓴 뒤, **저장된 절대경로**와 한 줄 요약을 보고하십시오.
- 추가 요청(특정 섹션 강조, 형식 변경 등)이 있으면 이 명령에 덧붙여 반영하십시오.
